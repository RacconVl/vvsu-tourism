# Инструкция по установке ВВГУ — Институт туризма

## Что вам понадобится

- Сервер или ВМ с Ubuntu 20.04 / 22.04 / 24.04 (рекомендуется)
- Минимум 1 GB RAM, 10 GB свободного места
- Открытый порт 80 (HTTP) в файрволе/облаке

---

## Шаг 1 — Установка Docker

Выполните по одной команде в терминале сервера:

```bash
# Обновить список пакетов
sudo apt-get update

# Установить необходимые пакеты
sudo apt-get install -y ca-certificates curl

# Добавить официальный ключ Docker
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Добавить репозиторий Docker
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Установить Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Разрешить запуск Docker без sudo (выйдите и зайдите снова после этой команды)
sudo usermod -aG docker $USER
```

> **После `usermod`** закройте терминал и откройте заново, чтобы изменения применились.

Проверьте, что Docker работает:
```bash
docker --version
docker compose version
```

Должно вывести что-то вроде:
```
Docker version 26.x.x, build ...
Docker Compose version v2.x.x
```

## Шаг 2 — Загрузка файлов проекта на сервер

### Вариант А — через SCP (с вашего компьютера)
```bash
# Выполняйте на СВОЁМ компьютере, не на сервере
scp vvsu-tourism.zip user@IP_СЕРВЕРА:/home/user/
```

### Вариант Б — загрузить через браузер, затем скачать на сервер
Загрузите архив на любой файлообменник (например, files.fm, mega.nz),
затем на сервере:
```bash
wget "ССЫЛКА_НА_АРХИВ" -O vvsu-tourism.zip
```

## Шаг 3 — Распаковка архива

На сервере:
```bash
# Установить unzip если нет
sudo apt-get install -y unzip

# Распаковать
unzip vvsu-tourism.zip

# Перейти в папку проекта
cd vvsu-tourism
```

## Шаг 4 — Создание файла .env

```bash
# Скопировать шаблон
cp .env.example .env

# Открыть для редактирования
nano .env
```

Файл `.env` выглядит так:
```
DATABASE_URL=postgresql://vvsu:vvsupassword@db:5432/vvsu_tourism
SESSION_SECRET=замените-на-случайную-строку
```

**Важно:**
- `DATABASE_URL` — оставьте как есть (Docker сам поднимет базу)
- `SESSION_SECRET` — замените на любую длинную случайную строку

Чтобы сгенерировать надёжный секрет:
```bash
cat /dev/urandom | tr -dc 'a-zA-Z0-9' | fold -w 48 | head -n 1
```
Скопируйте вывод и вставьте вместо `замените-на-случайную-строку`.

Сохраните файл: **Ctrl+O → Enter → Ctrl+X**

## Шаг 5 — Запуск

```bash
# Запустить все контейнеры в фоновом режиме
docker compose up -d --build
```

Первый запуск займёт **5–15 минут** — Docker скачивает образы и собирает проект.

Следите за процессом:
```bash
docker compose logs -f
```
Нажмите **Ctrl+C** чтобы выйти из просмотра логов (контейнеры продолжат работать).

NAME            STATUS

vvsu-tourism-web-1   Up

```bash

docker compose down

docker compose up -d --build
```

## Типичные ошибки и решения

### ❌ `permission denied while trying to connect to the Docker daemon`
```bash
# Добавьте себя в группу docker и перезайдите
sudo usermod -aG docker $USER
# Закройте и откройте терминал заново
```

### ❌ `port 80 is already allocated` или `bind: address already in use`
На сервере уже что-то занимает порт 80 (Apache, nginx, другой сервис).
```bash
# Узнать что занимает порт
sudo lsof -i :80
# или
sudo ss -tlnp | grep ':80'

# Остановить, например, nginx
sudo systemctl stop nginx
sudo systemctl disable nginx

# Снова запустить проект
docker compose up -d
```

### ❌ `no space left on device`
Закончилось место на диске.
```bash
# Проверить место
df -h

# Очистить неиспользуемые Docker-образы
docker system prune -a
```

### ❌ Контейнер `api` постоянно перезапускается
```bash
# Посмотреть логи ошибки
docker compose logs api --tail=50
```

Чаще всего причина — ошибка в `.env` или база данных ещё не готова.
Подождите 30 секунд и проверьте снова.

### ❌ Сайт открывается, но показывает ошибку подключения к API
```bash
# Проверить что API работает
curl http://localhost:8080/api/healthz
```

Должен вернуть `{"status":"ok"}`. Если нет — смотрите логи:
```bash
docker compose logs api
```

### ❌ Сайт не открывается по IP (таймаут)
Проверьте файрвол:
```bash
# Для Ubuntu с ufw
sudo ufw allow 80/tcp
sudo ufw status
```

Если используете облако (Yandex Cloud, VK Cloud, AWS и др.) —
откройте порт 80 в настройках группы безопасности / firewall rules в панели управления.

## Добавление домена (необязательно)

Если у вас есть домен, направьте A-запись на IP сервера, затем:

```bash
# Установить Certbot для HTTPS
sudo apt-get install -y certbot

# Получить SSL-сертификат (замените example.com на ваш домен)
sudo certbot certonly --standalone -d example.com
```

Затем добавьте в `nginx.conf` блок с SSL и пересоберите:
```bash
docker compose up -d --build web
```
